### 작업 규칙
- [이슈 생성](https://github.com/orgs/jungle-royale/projects/1/views/3)
- 이슈 이름으로 feature branch 생성
- 작업 후 PR
- merge 완료 후 issue 종료 및 브랜치 삭제

### 커밋 규칙
- 커밋 메시지 구조
  ```
  Type : Subject
  
  body
  
  footer
  ```
- 커밋 종류
  ```
  - feat 		: 새로운 기능 추가
  - fix 		: 버그 수정
  - docs 		: 문서 수정
  - refactor 	: 코드 리팩토링
  - test 		: 테스트 코드, 리팽토링 테스트 코드 추가
  - chore 	: 빌드 업무 수정, 패키지 매니저 수정
  ```
